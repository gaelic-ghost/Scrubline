# Security Policy

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting for security issues. Do
not include live credentials, tokens, private logs, or personal information in
an issue, discussion, pull request, or public fixture.

When a reproduction needs sensitive input, replace it with a synthetic value
that preserves only the format needed to demonstrate the problem.

Scrubline recognizes representative formats and should be treated as one
privacy layer, not a guarantee that every secret format will be detected.
